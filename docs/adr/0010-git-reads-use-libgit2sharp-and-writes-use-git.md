# Git reads use LibGit2Sharp and writes use Git

DotNetDo uses LibGit2Sharp for repository discovery and live structured reads, adapting shallow values such as repository-relative paths into DotNetDo types while exposing rich library types such as commits, branches, tags, status flags, and the underlying repository where useful. Repository mutations use repository-bound, awaitable Git commands executed through `Do.Exec`: `Add`, whole-file `Reset`, `Commit`, `Push`, annotated `CreateTag`, and `PushTag`; `GitRepository.Exec(arguments, options)` remains the repository-bound raw escape hatch. Using the Git executable for writes preserves normal hooks, signing, credentials, configuration, validation, and failure behavior rather than duplicating those policies in DotNetDo; merge, rebase, conflict handling, history-moving reset, chunk staging, and lower-level Git operations remain outside the convenience API.
