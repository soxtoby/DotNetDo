# Tool commands use awaitable record values

External command-line tool helpers use immutable record values instead of mutable fluent builders. Tool namespaces expose default command values as static fields, so scripts can write `await Tools.DotNet.Build with { NoRestore = true }`; awaiting a tool command executes it through `Do.Exec` and uses the same success/failure semantics as awaiting an `ExecProcess`. Properties own their argument-slot writes in their accessors, rendering remains slot-based, and shared dotnet option groups remain public non-generic base records because the CLI shares those options across commands.
