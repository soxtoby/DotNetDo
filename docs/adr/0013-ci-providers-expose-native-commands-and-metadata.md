# CI providers expose native commands and metadata

DotNetDo exposes process-stable nullable `Do.GitHubActions` and `Do.AzurePipelines` singletons, replacing `BuildEnvironment`: each exists only when its provider marker is active, exposes that provider's full documented non-deprecated runner-command surface as immediate methods, and snapshots all documented default metadata into typed provider-grouped values. The APIs deliberately preserve provider vocabulary and semantics rather than offering a portable `Do.CI` abstraction or public base type; shared command emission, escaping, environment reading, and write serialization remain internal. Native commands write directly to stdout or required runner files, fail when a required command-file variable is absent, and apply only shape validation; existing Serilog CI output delegates to the same emitters and, when both providers are active, emits both native annotations but ordinary output once. Resolving a `Do.Secret` additionally invokes every active provider's masking command, while calling a native masking method does not register a DotNetDo secret. Metadata includes build, runner, deployment, and classic-release variables where documented, uses natural path, boolean, numeric, URI, and timestamp types, and leaves unavailable values null; GitHub event payload parsing is outside this boundary.
