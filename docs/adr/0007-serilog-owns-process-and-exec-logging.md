# Serilog Owns Process And Exec Logging

DotNetDo configures Serilog's global logger from a module initializer when it is still the default silent logger. The bootstrap logger uses an `Information` minimum, `WriteTo.DefaultOutput()`, and `CreateRedactingLogger()`, and DotNetDo disposes only that logger at process exit. Tasks may replace `Log.Logger`; `WriteTo.DefaultOutput()` preserves the standard Serilog console sink locally and selects build-agent-native Azure Pipelines or GitHub Actions output from environment detection, while `CreateRedactingLogger()` wraps the completed logger so every sink receives events with resolved `Do.Secret` values redacted throughout templates, exceptions, property names, and nested values.

`Do.Exec` combines stdout and stderr lines into replayable typed `ExecOutput` objects in observed order, replacing ADR 0003's separate line, chunk, and text outputs. Its default logging sends stdout to Serilog at `Information` and stderr at `Error`; tasks may provide `ExecOptions.Log` to choose another destination or level, while a missing or `null` action uses the default. Command start and successful completion are `Debug`, while non-zero completion is `Error`. Captured output and public result/exception command data remain raw; the redacting logger masks known secrets before its sinks receive events. Redaction covers raw, JSON-escaped, and URI-escaped secret forms, but is an additional safeguard rather than a security boundary.
