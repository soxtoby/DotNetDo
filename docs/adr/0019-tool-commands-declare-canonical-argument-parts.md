# Tool commands declare canonical argument parts

Tool-command records store semantic values in their own properties and declare every canonically ordered rendered command part, making output independent of property assignment order. `ToolCommand` retains raw trailing additional arguments, execution behavior, and protected `Arg` and `Args` rendering helpers; boolean `Arg` overloads render optional or two-way switches. Argument prefixes contain no boundary whitespace: values attach to prefixes ending in `:`, `=`, or `,`, and otherwise receive one separating space. It no longer stores a separate command prefix, keyed argument slots, or setter/getter helpers. Concrete commands snapshot caller-owned collections, shared option bases expose ordered groups for explicit placement, and cross-property validation remains render-time. This supersedes ADR 0017's argument-slot storage and ordering while retaining its quote-by-default contract.
